## Derivation method

Standard Bitcoin/Cosmos hash160 from a secp256k1 public key:

```
1. SEC1 compressed pubkey[33]:
   prefix = 0x02 if y is even (y & 1 == 0)
            0x03 if y is odd  (y & 1 == 1)
   compressed = prefix || pubkey_x[32]

2. SHA-256(compressed[33]) → sha256_out[32]

3. RIPEMD-160(sha256_out[32]) → hash160[20]
```

This is the derivation that `derive_hash160_cosmos` implements and that the Babylon snapshot
generator (`scripts/generate-m2-snapshot.ts`) must use after ADR-038.

---

## Verification approach

Test vectors were generated by direct secp256k1 scalar multiplication (no library) using
secp256k1 parameters (`p`, `a=0`, `b=7`, `Gx`, `Gy` from SEC2), then `hashlib.sha256` +
`hashlib.new('ripemd160')` from Python stdlib. Both hash algorithms were validated against
canonical NIST test vectors before use:
- `RIPEMD-160("")` = `9c1185a5c5e9fc54612808977ee8f548b2258d31` ✅
- `RIPEMD-160("abc")` = `8eb208f7e05d987a9b044a8e98c6b087f15a0bfc` ✅

The G test vector (k=1) cross-checks against the Bitcoin canonical address derivation:
- Bitcoin address for privkey 1 (compressed): `1BgGZ9tcN4rm9KBzDn7KprQz87SZ26SAMH`
- Decoded hash160 = `751e76e8199196d454941c45d1b3a323f1433bd6` ✅ matches G vector below

## Snapshot generator companion

The `scripts/generate-m2-snapshot.ts` file must also be updated alongside the Noir circuit.
After ADR-038 is applied, `derive_hash160_sha256d_ts` (if present) must be replaced with:

```typescript
import { createHash } from 'crypto';

/**
 * Derives the standard Cosmos hash160 from a secp256k1 public key.
 * Matches the in-circuit derive_hash160_cosmos() in contracts/src/merkle.nr.
 * hash160 = RIPEMD-160( SHA-256( SEC1_compressed_pubkey[33] ) )
 */
function deriveHash160Cosmos(pubkeyX: Buffer, pubkeyY: Buffer): Buffer {
    // SEC1 compressed: prefix 0x02 if y even, 0x03 if y odd
    const prefix = (pubkeyY[31] & 1) === 0 ? 0x02 : 0x03;
    const compressed = Buffer.concat([Buffer.from([prefix]), pubkeyX]);  // 33 bytes

    const sha256Out = createHash('sha256').update(compressed).digest();
    const hash160   = createHash('ripemd160').update(sha256Out).digest();
    return hash160;  // 20 bytes
}
```

**Coordination required:** Both changes (circuit + snapshot generator) must be deployed
together. A snapshot built with SHA-256d and a circuit using Cosmos hash160 will fail to
verify any Merkle proofs (address mismatch). See ADR-038 §4 (coordinated upgrade plan).
