### 4b. New function: derive_hash160_cosmos()

Add immediately after `derive_hash160_sha256d()` — keep the sha256d fallback for reference:

```noir
// Derive a 20-byte Cosmos/Bitcoin hash160 from a secp256k1 public key using
// the PRODUCTION derivation: ripemd160(sha256(compressed_pubkey[33])).
//
// This matches the standard Cosmos (bbn1…) and Bitcoin (1…) address derivation:
//   1. Build compressed pubkey: [0x02|0x03] || pubkey_x[32]
//      Prefix is 0x02 if y is even (low bit of y[-1] = 0), else 0x03.
//   2. SHA-256(compressed[33]) → 32 bytes
//   3. RIPEMD-160(sha256_output[32]) → 20 bytes
//
// Replaces derive_hash160_sha256d() (the SHA-256d prototype fallback).
// The snapshot generator (synthetic-snapshot.ts) MUST use the same derivation.
//
// ADR: docs/adr-038-m2-ripemd160-upgrade-2026-07-02.md
// Test vectors: docs/adr-038-implementation-ready-2026-07-02.md §2
pub fn derive_hash160_cosmos(pubkey_x: [u8; 32], pubkey_y: [u8; 32]) -> [u8; 20] {
    // 1. Determine SEC1 compression prefix.
    //    Even y → 0x02; odd y → 0x03.
    //    "Even" = low bit of y's least-significant byte = 0.
    let prefix: u8 = if (pubkey_y[31] & 1) == 0 { 0x02 } else { 0x03 };

    // 2. Build compressed pubkey: [prefix] || pubkey_x[32] = 33 bytes.
    let mut compressed = [0u8; 33];
    compressed[0] = prefix;
    for i in 0..32 {
        compressed[i + 1] = pubkey_x[i];
    }

    // 3. SHA-256(compressed[33]).
    let sha256_output: [u8; 32] = sha256_var(compressed, 33);

    // 4. RIPEMD-160(sha256_output[32]) → 20-byte hash160.
    ripemd160(sha256_output)
}
```

## 7. synthetic-snapshot.ts change

The snapshot generator `scripts/synthetic-snapshot.ts` derives hash160 using the sha256d fallback. For the production snapshot (real BABY holders):

```typescript
// BEFORE (prototype fallback):
function deriveHash160Sha256d(pubkeyX: Buffer, pubkeyY: Buffer): Buffer {
    const inner = sha256(Buffer.concat([pubkeyX, pubkeyY]));
    const outer = sha256(inner);
    return outer.subarray(12, 32); // last 20 bytes
}

// AFTER (Cosmos production — must match circuit):
function deriveHash160Cosmos(pubkeyX: Buffer, pubkeyY: Buffer): Buffer {
    const prefix = (pubkeyY[31] & 1) === 0 ? 0x02 : 0x03;
    const compressed = Buffer.concat([Buffer.from([prefix]), pubkeyX]); // 33 bytes
    const sha256Inner = sha256(compressed);
    return ripemd160(sha256Inner); // 20 bytes — use 'ripemd160' npm package
}
```

**npm dependency to add:**
```
npm install ripemd160
```

The synthetic snapshot already derives fake keys — once RIPEMD-160 is in the circuit, the snapshot generator must produce matching leaf format (sha256(hash160_cosmos[20] || balance_be[8])).
